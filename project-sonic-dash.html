Project Overview

Sonic Dash is a console-based endless runner built in C++ that draws inspiration from the mobile Sonic Dash game. The player controls a character running at increasing speed, avoiding obstacles and collecting points. As speed increases, reaction time shrinks, creating a compelling difficulty curve.

This project was developed as part of a programming course to demonstrate mastery of object-oriented programming principles, game loop design, and collision detection in C++.

Key Features

⚡
Progressive Speed System

Game speed increases over time, scaling the challenge and keeping players engaged.

🔲
Obstacle Generation

Procedurally generated obstacles ensure each run feels unique and unpredictable.

💯
Score Tracking

Real-time score display with a persistent high-score system saved between sessions.

🎯
Collision Detection

Precise hitbox-based collision detection ensures fair, responsive gameplay.

Technical Implementation

The game is structured around core OOP principles, with separate classes handling the player, obstacles, game state, and rendering.

// Simplified game loop structure class Game { private: Player player; ObstacleManager obstacles; ScoreManager scores; float speed = 1.0f; bool running = true; public: void run() { while (running) { update(); render(); checkCollisions(); increaseSpeed(); } } void update() { player.move(); obstacles.generate(speed); scores.increment(speed); } };

Challenges & Learnings

Building a game loop in a console environment required careful management of timing and rendering. Key challenges included implementing smooth character movement with keyboard input without causing console flicker, designing a difficulty curve that felt challenging but not frustrating, and managing memory efficiently with dynamic obstacle generation.

This project deepened my understanding of game architecture patterns, real-time input handling, and the fundamentals of object-oriented design in C++.
