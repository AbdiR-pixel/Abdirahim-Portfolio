Project Overview

This project simulates a real-world university course registration system, built to demonstrate mastery of data structures in a practical context. The system handles student enrollment, course management, prerequisite checking, and waitlist management.

Developed as a final project for a Data Structures and Algorithms course, this system required careful selection of the right data structure for each component to maximize efficiency and correctness.

Data Structures Used

📋 Linked Lists

Used for managing dynamic student enrollment lists per course, allowing efficient insertions and deletions as students register or drop courses.

🌳 Binary Search Tree

Used for indexing courses by code, enabling O(log n) course lookups and efficient alphabetical traversal of the course catalog.

⏳ Priority Queue

Powers the waitlist system, prioritizing students by seniority (year of study) and registration timestamp when course spots open up.

🗂️ Hash Table

Provides O(1) average-case student lookup by ID, essential for fast authentication and profile retrieval in a high-traffic registration system.

System Features

  • Student registration and authentication by university ID
  • Course catalog with prerequisites and capacity management
  • Prerequisite validation before enrollment
  • Priority-based waitlist management
  • Automatic enrollment from waitlist when spots open
  • Student transcript generation showing enrolled courses
